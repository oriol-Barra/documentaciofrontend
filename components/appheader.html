Documentació del Component: AppHeader

Descripció

Aquest component carrega el capçal (header) de l'aplicació. Conté un enllaç de navegació cap a la pàgina d'inici i una sèrie de botons per accedir a diferents seccions com el perfil, el dashboard, les estadístiques, i més, en funció de si l'usuari està autenticat. A més, si l'usuari és administrador, es poden mostrar més opcions com crear lligues o veure els resultats de partides.

Propietats

isAuthenticated

Descripció: Estat per verificar si l'usuari està autenticat.

Tipus: Boolean

Valor per defecte: false

isAdmin

Descripció: Estat per verificar si l'usuari és administrador.

Tipus: Boolean

Valor per defecte: true

menuIsOpen

Descripció: Estat que determina si el menú està obert o tancat.

Tipus: Boolean

Valor per defecte: false

Mètodes

checkAuthentication

Descripció: Comprova si l'usuari està autenticat revisant si existeix un token a `localStorage`. Actualitza l'estat de `isAuthenticated` segons el token trobat.

Tipus: Mètode

checkIsAdmin

Descripció: Comprova si l'usuari és administrador revisant el valor de `admin` a `localStorage`. Actualitza l'estat de `isAdmin` basant-se en el valor trobat a `localStorage`.

Tipus: Mètode

logout

Descripció: Realitza la desconnexió de l'usuari eliminant el token d'autenticació de `localStorage`, enviant una sol·licitud de logout a l'API i redirigint a la pàgina de login.

Tipus: Mètode

Comportament i Funcionalitat

El component `AppHeader` es mostra en totes les pàgines per proporcionar una interfície consistent. Si l'usuari està autenticat, es mostra un menú amb diverses opcions (dashboard, perfil, estadístiques, etc.). Si l'usuari és administrador, es poden mostrar més opcions com crear lligues o veure els resultats de partides. En cas que l'usuari no estigui autenticat, es mostren botons per registrar-se o iniciar sessió.
